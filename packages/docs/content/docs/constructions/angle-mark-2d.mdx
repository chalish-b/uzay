## What is Angle Mark 2D?

`angleMark2D` draws the little arc that marks an angle: given a `vertex` and the two points its arms point at, it renders a small arc between them and computes the angle's measure. By default it marks the smaller angle, the short way between the arms; the `sweep` option chooses a different one. It is useful for labelling angles in geometry figures, triangles, and circle diagrams.

The construction only draws the mark. It does not draw the arms, the vertex, or any label. Those are scene-specific choices you build with regular items, passing the same points into the construction.

<AngleMark2DDemo />

The mark is an `angleMark2D` at a fixed vertex, with two draggable `point2d` arms driving its `a` and `b`. The readout reads the construction's `measure` atom (converted to degrees), so it always matches the drawn arc: drag an arm and both move together.

## Basic Usage

```typescript
import { Scene2D, angleMark2D, vec2 } from "uzay";

const scene = new Scene2D();

scene.create("camera2d", { center: vec2(0, 0), zoom: 1.5 });

const vertex = vec2(0, 0);
const a = vec2(2, 0);
const b = vec2(1, 2);

// Draw the arms yourself.
scene.create("line2d", { start: vertex, end: a });
scene.create("line2d", { start: vertex, end: b });

// Mark the angle between them.
const mark = angleMark2D(scene, { vertex, a, b, radius: 0.5, color: "gold" });
```

Under the hood the mark is a [circle2d](/docs/2d-items/circles) arc centered at the vertex, so it is a bare arc with no fill.

## Reactive Arms

The vertex and arms can be atoms, so the mark follows points you control. They are read-only as far as the construction is concerned: it never writes them, it only draws the angle they form.

```typescript
const a = scene.create("point2d", { coords: vec2(2, 0), color: "gold" });
const b = scene.create("point2d", { coords: vec2(1, 2), color: "gold" });

angleMark2D(scene, {
  vertex: vec2(0, 0),
  a: a.coords,
  b: b.coords,
});
```

Drag either point and the mark sweeps to match.

## Reading the Measure

The construction returns the angle as `measure`, a derived atom in **radians**. It is the same value used to draw the arc, so a readout built from it always matches the picture:

```typescript
const mark = angleMark2D(scene, { vertex, a, b });

scene.create("overlay2d", {
  position: vertex,
  content: scene.atom((get) => {
    const deg = (get(mark.measure) * 180) / Math.PI;
    return `${deg.toFixed(1)}^\\circ`;
  }),
  format: "latex",
  offset: vec2(0, -24),
});
```

By default `measure` reports the smaller angle, so it ranges from `0` to `π`. With other `sweep` modes it covers the angle actually drawn, up to `2π`.

## Choosing Which Angle: `sweep`

Two arms always cut the plane into two angles that sum to 360°. The `sweep` option decides which one the mark draws:

| `sweep` | Marks | Range |
|---------|-------|-------|
| `"minor"` (default) | The smaller angle | `0°`–`180°` |
| `"major"` | The larger angle | `180°`–`360°` |
| `"ccw"` | A counterclockwise sweep from `a` to `b` | `0°`–`360°` |
| `"cw"` | A clockwise sweep from `a` to `b` | `0°`–`360°` |

```typescript
angleMark2D(scene, { vertex, a, b, sweep: "major" });
```

`"minor"` and `"major"` pick by size, so as an arm swings past the straight-angle line the mark jumps to the other side, the one that is now smaller (or larger). For a static figure that is usually what you want.

`"ccw"` and `"cw"` instead commit to a turn direction. As an arm travels around the vertex the mark stays on one side and grows continuously from 0° through 180° to 360°, without ever swapping sides. Reach for these when the angle is animated or dragged through a straight angle and you want the arc to follow smoothly. `"ccw"` and `"cw"` mark opposite sides of the same pair of arms.

Since a major angle is never 90°, the right-angle square does not apply when `sweep` is `"major"`.

## Right Angles

By convention a right angle is marked with a square, so that happens on its own: whenever the angle is 90° the mark draws the square (with `radius` as its side) instead of the arc, and reverts to the arc otherwise. This is reactive, so an angle dragged through 90° picks up the square right at the corner and drops it again. Pass `squareRightAngle: false` to keep the arc at every angle.

```typescript
// Squares automatically at 90°; opt out to always show the arc:
angleMark2D(scene, { vertex, a, b, squareRightAngle: false });
```

## Styling

`radius` sets the size of the arc in world units; `color` and `thickness` style its stroke:

```typescript
angleMark2D(scene, {
  vertex,
  a,
  b,
  radius: 0.8,
  color: "tomato",
  thickness: 3,
});
```

For lower-level control, the returned `mark` is the underlying `circle2d`:

```typescript
mark.mark.strokeOpacity.set(0.6);
```

## Cleanup

Call `dispose()` to remove the mark from the scene:

```typescript
mark.dispose();
```

## API Reference

### Options

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `vertex` | `Vec2` | (required) | The angle's vertex |
| `a` | `Vec2` | (required) | A point the first arm points at |
| `b` | `Vec2` | (required) | A point the second arm points at |
| `radius` | `number` | `0.4` | Size of the mark in world units: the arc radius, or the square's side |
| `color` | `string` | `"white"` | Stroke color |
| `thickness` | `number` | `2` | Stroke width |
| `squareRightAngle` | `boolean` | `true` | Show a right-angle square (instead of the arc) whenever the angle is 90°; set false to keep the arc |
| `sweep` | `"minor" \| "major" \| "ccw" \| "cw"` | `"minor"` | Which of the two angles the mark draws: smaller, larger, or a fixed counterclockwise/clockwise sweep |

All options accept either a plain value or an atom.

### Returned Handle

| Field | Type | Description |
|-------|------|-------------|
| `mark` | `Circle2D` | The arc, as a `circle2d` |
| `measure` | `BoundAtom<number>` | The marked angle's measure in radians: `0` to `π` by default, up to `2π` with other `sweep` modes |
| `dispose` | `() => void` | Removes the mark from the scene |
