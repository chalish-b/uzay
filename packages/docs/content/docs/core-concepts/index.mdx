## The Three Pillars

### Scene

A **Scene** is a container that holds all your 3D objects (points, lines, curves, etc.) and manages their state. You create objects using `scene.create()` and they automatically become part of the scene.

```typescript
const scene = new Scene3D();
const point = scene.create("point3d", { coords: vec3(1, 2, 3) });
```

### View

A **View** connects a Scene to a DOM element and handles rendering. It uses Three.js under the hood to render the scene with WebGL. When anything in the scene changes, the view automatically re-renders.

```typescript
const view = new View3D(scene, camera.id, containerElement);
```

### Atoms

**Atoms** are reactive state containers. Any property of any object can be an atom, and when that atom's value changes, the scene automatically updates. This is the key to creating dynamic, interactive visualizations.

```typescript
const radius = scene.atom(2);
const point = scene.create("point3d", { radius: radius });

// Later: changing the atom updates the visualization
radius.set(5);
```
