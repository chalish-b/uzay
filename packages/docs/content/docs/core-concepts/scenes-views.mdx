## What is a Scene?

A **Scene** is the container that holds all your 3D items. Think of it as a stage where you place your mathematical visualizations. The scene manages:

- **Items**: All 3D items (points, lines, curves, axes, grids, cameras)
- **State**: Reactive atoms that drive dynamic updates
- **Invalidation**: Tracking what needs to re-render

Every Uzay application starts with creating a scene:

```typescript
import { Scene3D } from "uzay";

const scene = new Scene3D();
```

## Creating Items

Use `scene.create()` to add items to the scene. The first argument is the item type, and the second is an options object:

```typescript
// Create a point
const point = scene.create("point3d", {
  coords: vec3(1, 2, 3),
  color: "red",
  radius: 2,
});

// Create a line
const line = scene.create("line3d", {
  start: vec3(0, 0, 0),
  end: vec3(5, 5, 5),
  color: "blue",
});

// Create coordinate axes
const axes = scene.create("axes3d", {
  x: [-10, 10],
  y: [-10, 10],
  z: [-10, 10],
});
```

The `create()` method returns the created item, which you can use to access its reactive properties later.

## Removing Items

Remove items from the scene with `scene.remove()`:

```typescript
const point = scene.create("point3d", { coords: vec3(1, 1, 1) });

// Later: remove the point
scene.remove(point);
```

## What is a View?

A **View** is the renderer that displays a scene in the browser. It:

- Attaches to a DOM container element
- Uses Three.js for WebGL rendering
- Automatically re-renders when the scene changes
- Provides orbit controls for camera manipulation

## Creating a View

To render a scene, create a View with three arguments:

1. The scene to render
2. The ID of the camera to use
3. The DOM container element

```typescript
import { Scene3D, View3D, vec3 } from "uzay";

const scene = new Scene3D();

// You must create a camera first
const camera = scene.create("camera3d", {
  position: vec3(10, 10, 10),
  lookAt: vec3(0, 0, 0),
});

// Get the container element
const container = document.getElementById("canvas-container") as HTMLElement;

// Create the view
const view = new View3D(scene, camera.id, container);
```

The view will fill its container and automatically resize when the container size changes.

## Multiple Views

You can create multiple views of the same scene with different cameras:

```typescript
const scene = new Scene3D();

const frontCamera = scene.create("camera3d", {
  position: vec3(0, 0, 10),
  lookAt: vec3(0, 0, 0),
});

const topCamera = scene.create("camera3d", {
  position: vec3(0, 10, 0),
  lookAt: vec3(0, 0, 0),
});

const frontView = new View3D(scene, frontCamera.id, document.getElementById("front"));
const topView = new View3D(scene, topCamera.id, document.getElementById("top"));
```

Both views will update when the scene changes.

## Camera-scoped visibility

Multiple views become powerful when each one shows a different slice of the same scene. Every item carries a `tags` list, and every camera has a `visibleTags` list that decides which items it renders:

- When `visibleTags` is unset (`undefined`), the camera renders everything. This is the default, so a camera shows the whole scene until you give it tags to filter by.
- An item with no tags is universal: every camera renders it. Shared scaffolding like axes and grids can stay untagged and appear in every panel.
- Otherwise a camera renders an item when their tags intersect.

```typescript
const scene = new Scene3D();

// Each camera renders its own group, plus any untagged items.
const inputCam = scene.create("camera3d", {
  position: vec3(8, 6, 8),
  lookAt: vec3(0, 0, 0),
  visibleTags: ["input"],
});
const resultCam = scene.create("camera3d", {
  position: vec3(-8, 6, 8),
  lookAt: vec3(0, 0, 0),
  visibleTags: ["result"],
});

// Untagged: shows in both views.
scene.create("axes3d", { x: [-6, 6], y: [-6, 6], z: [-6, 6] });

// A draggable handle, rendered only by the input camera.
const handle = scene.create("point3d", {
  coords: vec3(2, 1, 0),
  draggable: "xyz",
  tags: ["input"],
});

// Derived from the handle, rendered only by the result camera.
scene.create("sphere3d", {
  center: vec3(0, 0, 0),
  radius: scene.atom((get) => {
    const c = get(handle.coords);
    return Math.hypot(c.x, c.y, c.z);
  }),
  tags: ["result"],
});
```

Because there is one scene and one store, the two panels share reactive state automatically. Dragging the handle in the input view grows the sphere in the result view through the derived atom alone, with no code needed to keep the two panels in sync. Interaction honors the filter too: a camera cannot grab an item it does not render.

To render only untagged items, pass an empty list: `visibleTags: []`.

`visibleTags` is a reactive atom like any other field, so you can drive it from state to reveal a group, swap which group is shown, or animate a reveal over time.

Camera-scoped visibility works the same way in 2D: give a [Camera2D](/docs/2d-items/camera) a `visibleTags` list and tag your [2D items](/docs/2d-items) to match.

## Switching Cameras

Change the active camera of a view:

```typescript
view.changeActiveCam(topCamera.id);
```

## Orbit Controls

Views come with built-in orbit controls:

- **Left-click + drag**: Rotate the camera around the target
- **Right-click + drag**: Pan the camera
- **Scroll**: Zoom in/out

## 2D Scenes and Views

Everything above has a planar equivalent. A `Scene2D` holds 2D items and renders through a `View2D`, with the same `create`, `remove`, and `atom` methods and the same three-argument view constructor:

```typescript
import { Scene2D, View2D, vec2 } from "uzay";

const scene = new Scene2D();

const camera = scene.create("camera2d", {
  center: vec2(0, 0),
  zoom: 1,
});

const container = document.getElementById("canvas-container") as HTMLElement;
const view = new View2D(scene, camera.id, container);
```

The only difference in feel is the controls: instead of orbiting, a `View2D` pans when you drag the background and zooms when you scroll. See [Camera2D](/docs/2d-items/camera) for framing and [2D Items](/docs/2d-items) for what you can place in the scene.

## API Reference

### Scene3D

| Method | Signature | Description |
|--------|-----------|-------------|
| `create` | `create(kind, options)` | Create a new item in the scene |
| `remove` | `remove(item)` | Remove an item from the scene |
| `atom` | `atom(value)` or `atom((get) => value)` | Create a reactive atom |

### View3D

| Constructor | Description |
|-------------|-------------|
| `new View3D(scene, cameraId, container)` | Create a view rendering `scene` using the camera with `cameraId` into `container` |

| Method | Signature | Description |
|--------|-----------|-------------|
| `changeActiveCam` | `changeActiveCam(cameraId)` | Switch to a different camera |

### Scene2D and View2D

`Scene2D` and `View2D` are the planar counterparts. They expose the same methods and constructor as the 3D versions: `new View2D(scene, cameraId, container)` renders a 2D scene, and `view.changeActiveCam(cameraId)` switches the active [Camera2D](/docs/2d-items/camera).
