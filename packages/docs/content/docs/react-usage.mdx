Uzay provides a React wrapper (`uzay/react`) that lets you declare scenes and items as JSX components. It handles the View3D lifecycle, cleanup, and camera management automatically.

## Quick Start

```tsx
import { Scene3D, vec3 } from "uzay";
import { Scene3DView, Camera3D, Point3D, Axes3D } from "uzay/react";

const scene = new Scene3D();

function App() {
  return (
    <Scene3DView scene={scene} style={{ width: "100%", height: 500 }}>
      <Camera3D position={vec3(5, 5, 5)} lookAt={vec3(0, 0, 0)} />
      <Point3D coords={vec3(1, 2, 1)} color="crimson" radius={3} />
      <Axes3D x={[-5, 5]} y={[-5, 5]} z={[-5, 5]} />
    </Scene3DView>
  );
}
```

`Scene3DView` creates a container div, initializes the Three.js renderer, and cleans everything up on unmount. Each item component (`Point3D`, `Line3D`, etc.) creates the corresponding scene object when it mounts and removes it when it unmounts.

## Available Components

Every scene object type has a matching React component:

| Component | Description |
|-----------|-------------|
| `Scene3DView` | Container that manages the View3D lifecycle |
| `Camera3D` | Camera with auto-registration |
| `Point3D` | 3D point / sphere marker |
| `Line3D` | Line segment between two points |
| `Vector3D` | Arrow vector |
| `Sphere3D` | Sphere at a given center |
| `Plane3D` | Infinite plane |
| `Axes3D` | Coordinate axes |
| `Grid3D` | Grid on a coordinate plane |
| `Overlay3D` | 2D text/LaTeX overlay anchored in 3D |
| `ParametricFunction3D` | Parametric curve |

Each component accepts the same options as `scene.create()` for that object type, plus event handler props (`onDrag`, `onClick`, `onHover`) and a `ref` prop.

## Reactive Props

There are two ways to make props reactive: plain React state and scene atoms.

### React State

For simple values that change occasionally (colors, sizes, toggles), regular React state works:

```tsx
import { useState } from "react";

function App() {
  const [color, setColor] = useState("red");

  return (
    <>
      <Scene3DView scene={scene} style={{ width: "100%", height: 500 }}>
        <Camera3D position={vec3(5, 5, 5)} lookAt={vec3(0, 0, 0)} />
        <Point3D coords={vec3(1, 2, 1)} color={color} />
      </Scene3DView>
      <button onClick={() => setColor(color === "red" ? "blue" : "red")}>
        Toggle Color
      </button>
    </>
  );
}
```

When a plain-value prop changes between renders, the component automatically syncs it to the item's internal atom. This goes through React's render cycle, which is perfectly fine for infrequent updates.

### Scene Atoms

For values that are shared across multiple items, change frequently (like during a drag), or need derived computations, use atoms:

```tsx
const scene = new Scene3D();
const pos = scene.atom(vec3(0, 0, 0));

// Derived atom: label text updates whenever pos changes
const label = scene.atom((get) => {
  const p = get(pos);
  return `(${p.x.toFixed(1)}, ${p.y.toFixed(1)}, ${p.z.toFixed(1)})`;
});

function App() {
  return (
    <Scene3DView scene={scene} style={{ width: "100%", height: 500 }}>
      <Camera3D position={vec3(5, 5, 5)} lookAt={vec3(0, 0, 0)} />
      <Point3D coords={pos} color="cyan" draggable="xyz" />
      <Overlay3D anchor={pos} content={label} />
    </Scene3DView>
  );
}
```

Atom props bypass React entirely. When `pos` changes (e.g., during a drag), the point and overlay update directly through the atom system with no React re-render. This is important for smooth, high-frequency updates.

### When to Use Which

| Use React state when... | Use atoms when... |
|--------------------------|-------------------|
| The value is simple and local | Multiple items share the same value |
| Updates are infrequent (button clicks, toggles) | Updates are frequent (drag, animation) |
| You don't need derived computations | You need computed/derived values |

You can mix both freely in the same component. For instance, use React state for a color toggle and atoms for a draggable position.

## The `useSceneAtom` Hook

If you prefer creating atoms inside your components (instead of at module scope), use the `useSceneAtom` hook. It works like `scene.atom()` but pulls the scene from context:

```tsx
import { Scene3DView, Camera3D, Point3D, useSceneAtom } from "uzay/react";
import { vec3 } from "uzay";

function App() {
  return (
    <Scene3DView style={{ width: "100%", height: 500 }}>
      <MyVisualization />
    </Scene3DView>
  );
}

function MyVisualization() {
  const pos = useSceneAtom(vec3(0, 0, 0));

  return (
    <>
      <Camera3D position={vec3(5, 5, 5)} lookAt={vec3(0, 0, 0)} />
      <Point3D coords={pos} color="gold" draggable="xyz" />
    </>
  );
}
```

Notice that `Scene3DView` has no `scene` prop here. When omitted, it creates a `Scene3D` internally, and `useSceneAtom` accesses it through context.

<Callout type="info" title="React Compiler">
  `useSceneAtom` is backed by `useMemo` to keep the atom stable across re-renders. If you're using
  [React Compiler](https://react.dev/learn/react-compiler), the memoization is handled automatically,
  so you can use `scene.atom()` directly with `useScene()` if you prefer.
</Callout>

`useSceneAtom` supports all three atom types:

```tsx
// Primitive atom
const radius = useSceneAtom(5);

// Derived atom (read-only)
const label = useSceneAtom((get) => `r = ${get(radius)}`);

// Writable atom (read + write)
const clamped = useSceneAtom(
  (get) => Math.min(get(radius), 10),
  (get, set, value: number) => set(radius, value),
);
```

<Callout type="warning">
  `useSceneAtom` must be called inside a child of `Scene3DView`, not in the same component that renders it.
  This is because it reads the scene from React context, which is only available to children.
</Callout>

## Event Handlers

Attach drag, click, and hover handlers as props:

```tsx
<Point3D
  coords={pos}
  color="cyan"
  draggable="xyz"
  onDrag={(event) => {
    if (event.phase === "move") {
      pos.set(event.worldPosition);
    }
  }}
  onClick={(event) => {
    console.log("Clicked at", event.worldPosition);
  }}
/>
```

These map directly to the imperative `.on()` method. The event objects are the same as documented in [Interactions](/docs/core-concepts/interactions).

## Camera

`Scene3DView` manages camera selection automatically.

### Single Camera

If there's one `Camera3D` child, it becomes the active camera:

```tsx
<Scene3DView scene={scene}>
  <Camera3D position={vec3(10, 10, 10)} lookAt={vec3(0, 0, 0)} />
  <Point3D coords={vec3(0, 0, 0)} />
</Scene3DView>
```

### Multiple Cameras

With multiple cameras, use the `active` prop to pick one:

```tsx
<Scene3DView scene={scene}>
  <Camera3D position={vec3(10, 10, 10)} lookAt={vec3(0, 0, 0)} active />
  <Camera3D position={vec3(0, 10, 0)} lookAt={vec3(0, 0, 0)} />
  <Point3D coords={vec3(0, 0, 0)} />
</Scene3DView>
```

If no `Camera3D` child is provided at all, `Scene3DView` creates a default camera internally.

### Reactive Camera Properties

Camera properties can also be atoms, just like any other item:

```tsx
const camPos = scene.atom(vec3(10, 10, 10));

<Camera3D position={camPos} lookAt={vec3(0, 0, 0)} />
```

For the full list of camera properties, see [Camera](/docs/objects/camera).

## Refs and Imperative Access

Use the `ref` prop to get the underlying item instance for imperative operations:

```tsx
import { useRef } from "react";
import type { Point3D as Point3DType } from "uzay";

function App() {
  const pointRef = useRef<Point3DType>(null);

  return (
    <>
      <Scene3DView scene={scene}>
        <Camera3D position={vec3(5, 5, 5)} lookAt={vec3(0, 0, 0)} />
        <Point3D ref={pointRef} coords={vec3(1, 2, 3)} color="red" />
      </Scene3DView>
      <button onClick={() => {
        const pt = pointRef.current;
        if (pt) {
          console.log("Position:", pt.coords.get());
          pt.color.set("blue");
        }
      }}>
        Log Position
      </button>
    </>
  );
}
```

The ref gives you the same object you'd get from `scene.create()`, so all atom fields and methods (`.on()`, `.off()`, etc.) are available.

### `useScene()` Hook

Use `useScene()` to access the `Scene3D` instance from any child of `Scene3DView`. This is useful for features that don't have React component equivalents yet:

```tsx
import { useScene } from "uzay/react";

function MyComponent() {
  const scene = useScene();

  // Full access to the imperative API
  // e.g., creating items dynamically, subscribing to atoms, etc.
  useEffect(() => {
    const item = scene.create("sphere3d", { center: vec3(0, 0, 0), radius: 1 });
    return () => scene.remove(item);
  }, [scene]);

  return null;
}
```

<Callout type="info">
  The React wrapper covers the most common use cases, but the imperative core is where most development
  effort goes. If you find something the wrapper doesn't support, `useScene()` gives you full access
  to the underlying `Scene3D` and its API.
</Callout>

## Complete Example

Here's a more complete example with reactive atoms, event handling, and UI controls:

```tsx
import { useState } from "react";
import { Scene3D, vec3 } from "uzay";
import {
  Scene3DView,
  Camera3D,
  Point3D,
  Line3D,
  Axes3D,
  Grid3D,
  Overlay3D,
} from "uzay/react";

const scene = new Scene3D();
const pos = scene.atom(vec3(1, 2, 0));
const label = scene.atom((get) => {
  const p = get(pos);
  return `(${p.x.toFixed(1)}, ${p.y.toFixed(1)}, ${p.z.toFixed(1)})`;
});

function App() {
  const [lineColor, setLineColor] = useState("dodgerblue");

  return (
    <div style={{ width: "100%", height: "100%" }}>
      <Scene3DView scene={scene} style={{ width: "100%", height: "100%" }}>
        <Camera3D position={vec3(8, 6, 8)} lookAt={vec3(0, 0, 0)} />

        <Point3D
          coords={pos}
          color="cyan"
          radius={3}
          draggable="xz"
          onDrag={(e) => {
            if (e.phase === "move") pos.set(e.worldPosition);
          }}
        />

        <Line3D start={vec3(0, 0, 0)} end={pos} color={lineColor} />
        <Overlay3D anchor={pos} content={label} format="text" />

        <Axes3D x={[-5, 5]} y={[-5, 5]} z={[-5, 5]} />
        <Grid3D plane="xz" />
      </Scene3DView>

      <button onClick={() => setLineColor(lineColor === "dodgerblue" ? "tomato" : "dodgerblue")}>
        Toggle Line Color
      </button>
    </div>
  );
}
```
